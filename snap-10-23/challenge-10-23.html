How Git Flow Branches Work

Master Branch

A shell is created for the project on this master branch, but no development is done on this branch. Merges into this branch only occur after each sprint. Code that will be deployed to the server.

Develop Branch

One person on the team creates the develop branch. This branch is pulled from the master branch. Where we merge feature branches.

Feature Branch

Each member of the project team creates a specific feature branch that they are working on from the develop branch and codes and commits. Merges to the develop branch from each feature branch only occurs when that branch is ready to be tested.

Hot Fix Branch

This is pulled from the master branch and is only created when there's a need for an important fix to the code.

Release Branch

This is pulled from the master branch once all feature branches have been tested and merged to the develop branch.
